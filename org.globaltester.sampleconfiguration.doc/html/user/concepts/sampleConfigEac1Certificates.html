EACv1 Certificates

The tab EACv1 Certificates allows to provide details on certificates needed for TerminalAuthentication v1.

The dropdown Choose which certificates to use shall be self-explanatory. Based on the selection here you need to either fill the next section or the one after that. You can select between either static certificates or generated certificates.

Selection of generated certificates above allows all testcases to be executed but requires a few steps for preparation. First you need to define the directory path where the required certificate chains shall be generated. This can be entered/selected in the field Certificates directory.
In order to make all required certificates available within that folder you need to configure appropriate root certificates within the preferences (See here for details). Afterwards you need to run the generation scripts from your script bundle.

NOTE: In accordance with the implemented test specifications samples that support migration of TA cryptosystem need to be tested for each allowed target cryptosystem! This implies you need to run the whole test set again after the migration based on the new configuration. This new configuration will require you to build a new appropriate SampleConfig.

Selection of static certificates reduces the number of executable testcases (essentially only those with positive TA) but is still feasible if you can't provide root certificates/keys. The fields DV certificate and IS certificate shall point to the appropriate certificates. The field IS key shall point to the matching terminal private key.
