ePassport Application

The tab ePassport allows to provide details on supported features of the ePassport application.

The section Supported profiles allows to select whether or not the sample contains an ePassport application at all. If you don't check the box ePassport most other options on this tab are irrelevant.

The section Supported data groups allows to select which datagroups are present within the ePassport application on the sample.

The section Preferred parameters allows to configure access conditions. Preferred method to open ePassport defines how to establish secure messaging with the ePassport (if the testcase does not explicitly state what method to use). Protection of data groups for Passive Authentication defines how to establish the secure messaging channel for read access to datagroups within the testcases for Passive Authentication. Selecting EAC here will perform CA and TA before accessing datagroups in those testcases. APDU to test secure messaging shall contain an APDU that passes successfully when a valid secure messaging is established and fails otherwise. If not provided this defaults to 00 B0 81 00 00.

NOTE: In accordance with the implemented test specifications samples that support more than one opening method need to be tested using each of them! This implies you need to run the appropriate tests (with all parameters in the section Preferred parameters set) for different configurations.

The section Sample configuration files enables you to perform layer 7 tests without access to a physical sample. If you provide a directory here and tick the checkbox to use it datagroups are read from the directory and a card is not accessed at all.

The section PA Certificates contains configuration options required to perform Passive Authentication. Please provide the paths to the named certificates in the appropriate fields.
