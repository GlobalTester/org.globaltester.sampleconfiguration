Chip Authentication v3

The tab CAv3 allows to provide details needed for ChipAuthentication v3.

The section Protocol support allows to select whether or not the sample supports chip authentication in version 3 at all. If you don't check the box CA all other options on this tab are irrelevant.

The section Supported features describes which variants of chip authentication are supported according to the following descriptions:
Select KeyReuse if your sample reuses the ephemeral key from PACE-GM for CAv3.

The section Preferred parameters allows to provide needed input for the tests. The field KeyID allows to specify the key ID use for CAv2 during the tests.

NOTE: In accordance with the implemented test specifications samples that support more than one key/algorithm for chip authentication need to be tested using each of the keys! This implies you need to run the appropriate tests (with all parameters in the section Preferred parameters set) for different configurations.
