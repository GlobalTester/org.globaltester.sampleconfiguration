SampleConfiguration

A SampleConfiguration stores all relevant configuration data for a specific test sample (e.g. card, inspection system) in one place. TestCampaigns can be executed with different SampleConfigurations to test different test samples. A SampleConfiguration can be created by the corresponding wizard as described in this section.

The following screenshot shows an example sample configuration from GT Prove ePA. For example you can set the parameters "Platform ID" and "Sample ID". These values will be used within the reports and can be helpful to identify the reports later and to assocciate them with the different test situations.

The next screenshot exemplarily shows the tab "MRZ". Here you can define the MRZ of your sample.
