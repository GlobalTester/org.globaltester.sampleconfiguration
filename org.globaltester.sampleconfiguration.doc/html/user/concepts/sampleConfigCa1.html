Chip Authentication v1

The tab CAv1 allows to provide details needed for ChipAuthentication v1.

The section Protocol support allows to select whether or not the sample supports chip authentication in version 1 at all. If you don't check the box CA all other options on this tab are irrelevant.

The section Supported features describes which variants of chip authentication are supported according to the following descriptions:
KAT Chip authentication using MSE:SetKAT
ATGA Chip authentication using MSE:SetAT followed by GeneralAuthenticate
DH Chip authentication based on Diffie-Hellman
ECDH Chip authentication based on elliptic curve cryptography
KeyRef selection of CA key via key reference

The section Preferred parameters allows to provide needed input for the tests. The field Preferred KeyRef CA allows to specify a key reference that shall be used during the tests. The field Invalid KeyRef CA allows to specify a key reference that the chip will reject, this is used to check chip behavior when invalid key references are supplied. The checkbox Force use of preferred keyRef CA can be used to circumvent all automatic selection mechanisms for chip authentication key references.

NOTE: In accordance with the implemented test specifications samples that support more than one key for chip authentication need to be tested using each of the keys! This implies you need to run the appropriate tests (with all parameters in the section Preferred parameters set) for different configurations. Please make sure to set the force flag in this case.
