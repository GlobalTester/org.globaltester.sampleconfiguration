Restricted identification

The tab RI allows to provide details needed for Restricted Identification tests.

The section Protocol support allows to select whether or not the sample supports restricted identification at all. If you don't check the box RI all other options on this tab are irrelevant.

The section Supported features describes which variants of RI are supported according to the following descriptions:
DP sample provides optional RestrictedIdentificationDomainParameterInfo data structure

The section Preferred parameters allows to enforce use of specific keys for the tests. The field Key ID force the use of a specific key to be used. The field key is "authorized only" needs to be checked iff the key can only be used by authorized terminals.

NOTE: In accordance with the implemented test specifications samples that support more than one RI key need to be tested using each key! This implies you need to run the appropriate tests (with all parameters in the section Preferred parameters set) for different keys.
