PACE

The tab PACE allows to provide details needed for PACE during the test execution.

The section Protocol support allows to select whether or not the sample supports PACE at all. If you don't check the box PACE all other options on this tab are irrelevant.

The section Supported passwords allows to select which passwords (according to BSI TR-03110) are supported.

The section Supported features describes which variants of PACE are supported according to the following descriptions:
ECDH PACE based on elliptic curve cryptography
DH PACE based on Diffie-Hellman
GM PACE with generic mapping
IM PACE with integrated mapping
CAM PACE with chip authentication mapping

The section Preferred parameters allows to enforce use of specific parameters for the tests. The field OID allows to force the use of a specific PACE-OID to be used. This requires that the sample supports this OID according to EF.CardAccess. The field Domain Parameter allows to force the use of a specific PACE domain parameter set, by providing its id as HexString. This requires that the sample supports this domain parameters according to EF.CardAccess. The field Invalid Password ID needs to specify a password id that the chip will reject. The checkbox Force static binding between PACE and TAv1 must be checked iff the sample uses Documentnumber instead of PACE ephemeral key in TA.

NOTE: In accordance with the implemented test specifications samples that support more than one PACE configuration need to be tested using each of these! This implies you need to run the appropriate tests (with all parameters in the section Preferred parameters set) for different configurations.
