eSign Application

The tab eSIGN allows to provide details on supported features of the eSign application.

The section Protocol support allows to select whether or not the sample contains an eSign application at all. If you don't check the box eSIGN the other options on this tab are irrelevant.

The section Preferred parameters contains configuration options required to perform the relevant eSign test cases. eSign-PIN shall contain the value of the eSign PIN to be used. This value will be used to set the Pin during the tests and to use the eSign keys in later tests. eSign-PIN reference defines the password reference to use for eSign. DigitalSignatureTemplate shall contain the DigitalSignatureTemplate required for the key management function as HexString. This needs to contain at least the key reference.

NOTE: In accordance with the implemented test specifications the eSign tests cases heavily rely on each other. So either execute the whole test suite on a vanilla sample or have close look at the testcases and make sure all dependencies are met when executing them out of order.
